## 2. 교육 운영 흐름

### 12주 전체 흐름

```
[Phase 1: 기초] Week 1~5
  W1: AI 정의 + 선형회귀 (손계산 입문)
  W2: ANN + 역전파 (Chain Rule 핵심)
  W3: CNN (이미지)
  W4: RNN + LSTM (시계열)
  W5: ★ Phase 1 종합 시험 ★
       → 통과 못하면 반복

[Phase 2: 현대+실전] Week 6~11
  W6: Seq2Seq + Attention
  W7: Transformer
  W8: BERT 텍스트 분류
  W9: API 챗봇 웹사이트
  W10: YOLO Detection
  W11: ★ Phase 2 종합 시험 ★
        → 통과 못하면 반복

[마무리] Week 12
  W12: ★ 최종 발표 + 과제 투입 판단 ★
```

### 매주 운영 패턴

```
월: 이론 설명 (칠판/화이트보드)
    → 이전 모델의 한계 → 이번 모델의 해결 → 핵심 수식
    → 데모 시연 (동작하는 모습 먼저 보여주기)

화~수: 손계산 + 코딩 실습
    → 종이에 직접 계산
    → numpy 손코딩으로 검증
    → 프레임워크(Keras/PyTorch)로 구현

목: 다양한 데이터셋 실험
    → 기존 레포 코드(keras/, torch/) 참고하며 직접 돌려보기
    → 목표 수치 달성 시도

금: 숙제 마무리 + 체크리스트 점검
    → 통과 못한 항목 보충
    → 다음 주 예습
```

## 3. 평가 체계

### 3-1. 일상 평가 (매주)

**숙제 제출**: 매주 금요일까지
- GitHub repo에 push 또는 노션에 업로드
- 손계산은 사진 촬영하여 제출
- 코드는 실행 가능한 상태로

**체크리스트 점검**: 매주 월요일 시작할 때
- 지난주 통과 체크리스트 확인
- 통과 못한 항목 → 보충 후 재확인
- **전 주 체크리스트 미통과 시 다음 주 진행 불가**

### 3-2. 종합 시험 (W5, W11)

**Week 5: Phase 1 시험**

손코딩 시험 (45분):
| # | 문제 | 시간 | 배점 |
|---|------|------|------|
| 1 | Linear Regression 1 epoch | 5분 | 10점 |
| 2 | MLP 2층 Chain Rule | 10분 | 25점 |
| 3 | CNN 출력 크기 5문제 | 5분 | 15점 |
| 4 | RNN hidden state 3 step | 10분 | 25점 |
| 5 | LSTM gate 1 step | 15분 | 25점 |

코딩 시험 (80분):
| # | 문제 | 시간 | 합격 |
|---|------|------|------|
| 1 | 새 정형 → MLP 회귀 | 20분 | R² ≥ 0.7 |
| 2 | 새 이미지 → CNN 분류 | 30분 | acc ≥ 80% |
| 3 | 새 시계열 → LSTM 예측 | 30분 | MSE ≤ 0.05 |

통과: 손코딩 60점 이상 + 코딩 2/3 합격

**Week 11: Phase 2 시험**

실전 테스트:
| # | 문제 | 시간 | 합격 |
|---|------|------|------|
| 1 | 새 텍스트 → BERT 분류 | 30분 | F1 ≥ 0.8 |
| 2 | 챗봇 웹사이트 완성 | 1시간 | 대화+스트리밍 |
| 3 | 이미지 라벨링 → YOLO → 웹캠 | 2시간 | 탐지 동작 |

통과: 2/3 합격 + 문제풀이 12/15

## 4. 교육 운영 팁

### 4-1. 첫날 (Week 1 월요일)

**분위기가 중요하다.** 처음에 겁먹으면 12주 내내 위축됨.

1. 연구실 투어 + 장비 소개 (Jetson, GPU 서버 등)
2. 현재 과제에서 실제로 쓰이는 AI 데모 시연
3. "너희도 12주 후면 이걸 만든다" 라는 비전 제시
4. 선배들의 첫 학기 경험담 공유
5. 편하게 질문하라는 문화 강조

### 4-2. 손계산 지도법

- **처음에 느리면 정상.** 재촉하지 말 것
- 칠판에 교육자가 먼저 1 epoch 풀어보이고, 나머지는 학생이
- 계산 틀려도 **과정이 맞으면 인정** (숫자 실수 vs 개념 오류 구분)
- 손계산 결과를 바로 numpy로 검증 → "맞았다!" 성취감

### 4-3. 코딩 지도법

- 처음엔 기존 코드(keras/, torch/) 수정하는 것부터
- 빈 파일에서 시작하는 건 Week 5 시험부터
- 에러 났을 때 **바로 답 주지 말고** 에러 메시지 읽는 법부터 알려주기
- Stack Overflow / 공식 문서 찾는 습관 들이기

### 4-4. 진도가 밀릴 때

| 상황 | 대응 |
|------|------|
| 1~2명만 밀림 | 개별 보충 (선배 1:1 멘토링) |
| 전체적으로 밀림 | 해당 주차 1주 연장, 숙제 경량화 |
| 특정 주제만 어려움 | 추가 설명 세션 + 예제 추가 |
| W5 시험 불합격 | Phase 1 핵심만 1주 재복습 후 재시험 |

### 4-5. 주의사항

- **코드 복붙을 경계할 것**: 돌아가기만 하면 되는 게 아니라, 왜 이렇게 되는지 설명할 수 있어야 함
- **"왜?"를 항상 물어볼 것**: "LSTM 썼습니다" → "왜 LSTM이야? RNN은 왜 안 돼?"
- **비교하지 말 것**: 학생마다 속도가 다름. 느려도 12주 안에 들어오면 됨
- **실패해도 괜찮다는 분위기**: 모델 성능 안 나오는 것도 경험. 왜 안 나오는지 분석이 중요
