# 석사 과정생 트랙 — 논문 구현 & 커스터마이징 (3개월)

> 신입생 커리큘럼(12주)을 마친 후, 또는 기존 석사 과정생이 진행하는 트랙.
> 목표: **3개월 안에 논문을 읽고, 구현하고, 커스터마이징할 수 있는 수준**

---

## 신입생 커리큘럼과의 관계

```
[신입생 12주]                    [석사 트랙 12주]
모델 구현할 수 있다        →     논문을 구현하고 바꿀 수 있다
코드 돌릴 수 있다          →     실험을 설계하고 비교할 수 있다
"이 모델 뭐야?" 설명       →     "이 논문 뭐가 새로운 거야?" 설명
프레임워크 사용             →     논문 코드를 읽고 수정
데모 만들기                →     과제 요구사항 구현 + 보고서
```

---

## 전체 흐름

```
Month 1 (W1~4):  논문 읽기 + 재현 기초 + 실험 환경
Month 2 (W5~8):  논문 재현 + 커스터마이징 + 과제 도메인 심화
Month 3 (W9~12): 독립 연구 (논문 선정 → 재현 → 개선 → 발표)
```

# Month 1: 논문 읽기 + 재현 기초 (W1~4)

## W1: 논문 읽는 법 + 실험 환경 세팅

### 학습
- 논문 구조 이해: Abstract → Intro → Related Work → Method → Experiment → Conclusion
- 논문 읽는 순서: Abstract → Figure/Table → Conclusion → Method → 나머지
- "이 논문이 뭘 했는지"를 3줄로 요약하는 연습
- 어디서 논문을 찾는가: Google Scholar, arXiv, Papers With Code, Semantic Scholar

### 실습
- 실험 관리 환경 세팅
  - wandb (Weights & Biases) 계정 생성 + 사용법
  - Git branch 전략 (실험별 브랜치)
  - config 파일로 하이퍼파라미터 관리 (yaml/json)
  - random seed 고정, 재현 가능한 코드 구조

### 숙제
| # | 과제 | 제출물 |
|---|------|--------|
| 1 | 논문 3편 읽고 각각 3줄 요약 | 요약문 (논문 제목, 문제, 방법, 결과) |
| 2 | wandb 연동해서 MNIST CNN 학습 로그 기록 | wandb 대시보드 스크린샷 |
| 3 | 실험 코드 템플릿 만들기 (config, seed, logging 포함) | GitHub 코드 |

### 추천 논문 (첫 논문 읽기용, 짧고 영향력 큰 것)
- ResNet: "Deep Residual Learning for Image Recognition" (2015)
- Attention: "Attention Is All You Need" (2017)
- BERT: "BERT: Pre-training of Deep Bidirectional Transformers" (2018)

## W2: 첫 논문 재현 (간단한 것부터)

### 학습
- 논문 재현이란: 논문의 모델을 직접 구현해서 논문과 비슷한 성능을 내는 것
- Papers With Code에서 공식 코드 찾는 법
- 공식 코드를 읽는 법: README → config → model → train → eval 순서
- "코드가 없는 논문"은 어떻게 재현하는가

### 실습
- ResNet 재현 (CIFAR-10)
  - 논문 읽기 → 구조 이해 → PyTorch로 구현
  - 공식 코드와 비교
  - 논문의 Table과 비슷한 성능이 나오는지 확인

### 숙제
| # | 과제 | 제출물 |
|---|------|--------|
| 1 | ResNet-18을 PyTorch로 직접 구현 (torchvision 안 쓰고) | 코드 |
| 2 | CIFAR-10에서 학습, 논문 결과와 비교 | 성능 비교 표 |
| 3 | Skip Connection 빼면 성능이 어떻게 변하는지 실험 | 비교 결과 + 분석 |
| 4 | wandb에 실험 기록 | 대시보드 링크 |

### 통과 기준
- [ ] ResNet 구조를 코드로 구현할 수 있다
- [ ] 논문 Table의 성능과 ±3% 이내
- [ ] "Skip Connection이 왜 중요한지" 실험 결과로 설명할 수 있다

## W3: 논문 코드 읽기 + 수정하기

### 학습
- GitHub에서 논문 공식 코드를 클론해서 돌리는 프로세스
  - 환경 세팅 (requirements.txt, conda env)
  - 데이터 다운로드
  - 학습 실행
  - 결과 확인
- 남의 코드 읽는 법: 진입점(main.py) → config → model → data → train loop
- 코드 수정: backbone 교체, loss 변경, 데이터 교체

### 실습
- 감정 인식 관련 논문 1편의 공식 코드를 클론해서 돌리기
- 데이터셋을 다른 것으로 교체해서 돌려보기
- backbone을 다른 모델로 바꿔보기

### 숙제
| # | 과제 | 제출물 |
|---|------|--------|
| 1 | 감정 인식 논문 1편 선정, 공식 코드 클론 후 재현 | 실행 결과 + 논문 대비 성능 |
| 2 | 데이터셋을 다른 것으로 교체 후 결과 비교 | 비교 표 |
| 3 | 모델의 한 부분을 수정하고 결과 비교 | 수정 내용 + 비교 표 |

## W4: Ablation Study + 실험 보고서

### 학습
- Ablation Study란: 모델의 각 구성 요소를 하나씩 빼면서 기여도 측정
- 공정한 비교를 위한 실험 설계
  - 같은 데이터, 같은 split, 같은 epoch, 같은 seed
  - 여러 번 돌려서 평균 ± 표준편차
- 실험 결과를 표와 그래프로 정리하는 법
- 실험 보고서 작성법 (과제 보고서 양식 참고)

### 실습
- W2~3에서 한 실험들을 정리해서 Ablation Table 작성
- 실험 보고서 1편 작성 (A4 3~5 페이지)

### 숙제
| # | 과제 | 제출물 |
|---|------|--------|
| 1 | Ablation Table 작성 (최소 4개 변형 비교) | 표 |
| 2 | 실험 보고서 작성 | A4 3~5 페이지 (문제정의, 방법, 실험, 결과, 분석) |
| 3 | 5분 발표 | 발표 자료 |

### Month 1 통과 기준
- [ ] 논문을 읽고 3줄 요약할 수 있다
- [ ] 논문 코드를 클론해서 돌릴 수 있다
- [ ] 모델의 일부를 수정하고 결과를 비교할 수 있다
- [ ] Ablation Study 결과를 표로 정리할 수 있다
- [ ] 실험 보고서를 작성할 수 있다
