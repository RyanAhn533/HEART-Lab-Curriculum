# 석사 과정생 트랙 — 3개월 Condition (논문 구현 & 커스터마이징)

> 석사 입학 후 3개월 Condition 기간.
> 이 기간 안에 **논문을 읽고, 구현하고, 커스터마이징할 수 있는 수준**에 도달해야 한다.
> 매월 월간 발표에서 진행 상황을 발표하고 평가받는다.

---

## Condition 제도

```
입학/합류
  ↓
[Month 1] 논문 읽기 + 재현 기초    → ★ 1차 월간 발표
  ↓
[Month 2] 논문 재현 + 커스터마이징  → ★ 2차 월간 발표
  ↓
[Month 3] 독립 연구 + 과제 기여    → ★ 3차 월간 발표 (Condition 최종 평가)
  ↓
통과 → 정식 연구원으로 과제 투입
미통과 → 부족 영역 보충 후 재평가
```

- 매월 마지막 주에 **월간 발표** (전체 랩미팅에서)
- 학부 신입생도 동일하게 매달 발표 (신입생 커리큘럼 진행 상황)
- 발표 후 피드백 → 다음 달 계획에 반영

---

## 신입생 커리큘럼과의 관계

```
[신입생 12주]                    [석사 Condition 12주]
모델 구현할 수 있다        →     논문을 구현하고 바꿀 수 있다
코드 돌릴 수 있다          →     실험을 설계하고 비교할 수 있다
"이 모델 뭐야?" 설명       →     "이 논문 뭐가 새로운 거야?" 설명
프레임워크 사용             →     논문 코드를 읽고 수정
데모 만들기                →     과제 요구사항 구현 + 보고서
```

# Month 1: 논문 읽기 + 재현 기초 (W1~4)

## W1: 논문 읽는 법 + 실험 환경 세팅

### 학습

- 논문 읽는 순서: Abstract → Figure/Table → Conclusion → Method → 나머지
- "이 논문이 뭘 했는지" 3줄 요약 연습

### 실습

# 월간 발표 제도 (Condition 이후에도 계속)

Condition 3개월이 끝나도 **매달 월간 발표**는 계속한다.
학부 신입생, 석사 과정생 모두 동일.

## 월간 발표 운영

```
매월 마지막 주 (또는 첫째 주) — 전체 랩미팅에서 발표

발표 순서:
  1. 신입생: 이번 달 커리큘럼 진행 상황 (10분)
  2. 석사 Condition: 논문 재현/실험 진행 상황 (15~25분)
  3. 정식 연구원: 연구 진행 상황 (15분)
```

## 정식 연구원 월간 발표 양식

Condition 통과 후의 월간 발표:

| 순서 | 내용 | 시간 |
|------|------|------|
| 1 | 지난달 계획 대비 달성도 | 2분 |
| 2 | 이번 달 한 일 (실험 결과, 코드, 논문) | 8분 |
| 3 | 이슈 / 막힌 점 / 논의 사항 | 3분 |
| 4 | 다음 달 계획 | 2분 |

### 발표 자료에 포함할 것
- [ ] 지난달 계획 → 실제 달성 비교표
- [ ] 실험 결과 (새로 돌린 것)
- [ ] wandb / 실험 로그
- [ ] 이슈 사항 (있으면)
- [ ] 다음 달 구체적 계획 (주 단위)
